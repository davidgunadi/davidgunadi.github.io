Case Study

You are required to build a database, using any technology you want.
This database will receive 100 million records from various different vendors per hour.
The info is sent from GPS device (attached to a vehicle), and is received by a Kernel.
The info contains the GPS IMEI Number, Longitude, Latitude, Speed, and numerous other additional fields.

You would also need these raw data to provide "live view" for the end user from a website.
The live view would be shown over a map, showing all vehicle position, how fast it is currently going, and the last time the GPS information was received.
At any given time, 100 thousands end user will access your site, so performance is crucial.

Given the freedom to do anything you want, how would you design the database structure / technologies to ensure these following three things:
a. Performance
b. Cost and Efficiency
c. Scalability

Some things that you might consider are:
a. Should you create one database for all, or separate them based on Vendor?
b. Cloud / Local Server?
c. Would you consider using other technologies such as REDIS, etc?

Pengguna software anda, perlu melihat lokasi live 5000 unit ini dari website anda.
Pada saat yang sama, bisa 1000 user berbeda melihat lokasi live ini.

Anda juga harus menyediakan report bagi user seperti total jarak perjalanan kendaraan untuk jangka waktu tertentu.
Report kendaraan mana yang sedang berhenti, mana yang sedang berjalan, mana yang belum berjalan selama seminggu.

Dengan requirement diatas, bagaimana anda akan design database ini, dengan catatan bahwa performance dan efisiensi sangatlah penting.
Beberapa point yang harus anda jelaskan adalah:
a. Teknologi apa yang akan anda gunakan? (PostgreSQL, NoSQL, Redis, Index, Relation Table, REDIS, dll)
b. Apakah anda akan memisahkan GPS unit ke database berbeda jika unit ini melebihi angka tertentu? Misalnya sekarang masih 5000 unit, bagaimana kalo jadi 1 juta unit? Jelaskan plan anda jika ini terjadi.
c. Database schema yang sudah jadi harus di sediakan dan dipresentasikan saat interview.
